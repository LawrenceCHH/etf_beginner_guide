# ETF 新手教學網頁 — 專業編輯與 UI/UX 設計規劃書 (plan.md)

本規劃書旨在將 `guide.md` 的專業金融內容，重新打造成一個適合**新手、不擅長閱讀資料者**的「極速易讀、高質感、具備互動感」的響應式單頁網頁 (Single Page Application)。

---

## 一、核心編輯與設計理念

針對「不擅長閱讀資料者」與「投資新手」，我們採取以下三項核心策略：

1. **雙軌閱讀機制 (Dual-Track Reading Flow)**
   - **極速軌道 (Speed Track)**：預設僅呈現精煉的「編輯重點卡片」、「視覺化圖表」與「一目了然的比較表」，搭配大字體與加粗關鍵字，確保讀者在 2 分鐘內掃讀完核心觀念。
   - **深讀軌道 (Deep Track)**：每一章末尾提供流暢、帶有微動畫的「展開完整原文」折疊面板 (Accordion)，逐字保留 `guide.md` 原文，供想要深入鑽研細節的讀者隨時查閱，避免造成視覺超載。
2. **漸進式資訊揭露 (Progressive Disclosure)**
   - 拒絕一次性塞入大量文字，利用互動卡片、折疊面板、左右滑動步進器 (Stepper) 逐步引導使用者吸收。
3. **無 Emoji 向量美學 (Vector-based Aesthetics)**
   - 遵循專業 UI 設計規範，捨棄系統發散且缺乏質感的 Emoji，全站統一使用高雅的 **SVG 向量圖示 (如 Lucide Icons 風格)**，以高對比、流暢線條傳遞專業、安心的金融品牌感。

## 三、網站架構與內容重組規劃

網頁採用單一 `index.html` 架構，內嵌高質感 CSS 與 JS，確保零外部依賴、載入極速，完美相容 GitHub Pages。

### 1. Hero 首頁區塊 (視覺錨點)
- **文案**：一句話直擊新手痛點：「只要花少少的錢，就能讓全世界最賺錢的大公司幫你打工？3 分鐘輕鬆搞懂 ETF 與你的第一步！」
- **CTA 按鈕**：醒目的漸層按鈕「30 秒快速上手 🚀」，點選平滑捲動至第零章。
- **數據看板**：以大型數字呈現 4 個關鍵指標：`7-10%`（長期年化）、`100 元`（最低投資額）、`400 年`（股票歷史）、`5 大步驟`（輕鬆開始）。

### 2. 第零章 · 30秒快速上手 (懶人包)
- 將原本的 4 個 QA 點選，改造成 **FAQ 互動手風琴卡片**：
  - 「什麼是 ETF？」→ 展開顯示「綜合水果盤」比喻。
  - 「該選定存還是 ETF？」→ 展開顯示「防禦與進攻」的互補關係。
  - 「新手該買哪一檔？」→ 展開推薦「台股 0050、美股 VOO」雙台柱。
  - 「怎麼買？」→ 展開說明「開戶、扣款、安心生活」極簡三步。

### 3. 第一章 · 股票的前世今生 (時間軸)
- 設計一個**垂直時間軸 (Vertical Timeline)**：
  - **15-17世紀 (大航海時代)**：荷蘭東印度公司誕生，首發股票與交易所（痛點：船沉了就破產，解決方案：利益共享、有限責任）。
  - **18-19世紀 (泡沫與法規)**：牛頓也認賠的「南海泡沫」，促成證券交易委員會 (SEC) 與嚴格揭露法規的建立。
  - **20世紀至今 (現代全民時代)**：先鋒集團約翰·柏格推出首檔 S&P 500 指數型基金，開啟「買下整個市場」的長線投資革命。
- **底部折疊區**：`點擊展開第一章完整原文 📄`。

### 4. 第二章 · 商業本質與現金流動 (互動對照)
- **概念澄清**：設計兩個雙色對焦卡片區隔「次級市場交易（錢流向賣方投資人）」與「利息 vs. 股利（股東分紅）」的本質。
- **iPhone 金流流動圖表**：將原本的表格重新排版為**角色卡片流向組**：
  - *消費者* 💸 → *Apple 企業* 💰 → *長線投資人/ETF* 📈。
  - 加強「邏輯修正提醒」的視覺包裝，使用藍綠色背景框，強調投資非零和遊戲，而是與企業共榮。
- **底部折疊區**：`點擊展開第二章完整原文 📄`。

### 5. 第三章 · 從定存到投資 (風險與資產配置)
- **定存 vs 大盤型 ETF 比較**：採用雙欄對比卡片，分別塗裝為「防禦灰/藍」與「進攻綠」，強調功能互補性。
- **必須優先考慮定存的三大情境**：採用警告感十足的「黃/橙色警告卡片」（緊急預備金、短期大額支出、極度保守心態），體現客觀性。
- **ETF 波動風險提示**：使用紅色警示邊框 (Border-l-4 border-red-500) 呈現，提醒短期內可能有 20-30% 跌幅。
- **發行商 3 大獲利模式**：以「發行商是開賭場的服務商，而非賭客」為主題，以三張並列的小卡片呈現其獲利秘密（管理費、借券、換股折讓）。
- **底部折疊區**：`點擊展開第三章完整原文 📄`。

### 6. 第四章 · 主流 ETF 大解密 (屬性卡片對決)
- 將 0050、VOO、QQQ 設計為**三張大型對決卡片 (VS Cards)**：

  - **VOO Vanguard標普500**：全球五百強、歷史穩健、美元換匯。
  - **QQQ Invesco納斯達克100**：科技 AI 巨頭、高爆發力、波動度高。
  - 每張卡片內含「追蹤市場」、「最大優點」、「最大風險」與「適合人群」的精簡指標。
- **底部折疊區**：`點擊展開第四章完整原文 📄`。

### 7. 第五章 · 實戰演練 (通用步進器與國泰證券實戰)
- **第一區塊 (通用定期定額流程)**：將 7 個通用步驟設計成一個**互動式步進器 (Interactive Stepper Layout)**，每個步驟包含互動 Checkbox 供讀者點擊勾選。
- **第二區塊 (國泰證券 App 實戰步驟)**：以台股 0050 申購為例，提供國泰證券 2026 最新定期定額 5 步驟圖文實戰步驟（包含進入專區、搜尋標的、設定金額與每月交易日、確認申購內容、申請中畫面與扣款準備），引用您上傳的 5 張本地實戰步驟截圖，並標註國泰官方來源網址。
- **最後的叮嚀**：以質感金句卡片的形式收尾，強調紀律與時間的複利力量。
- **底部折疊區**：`點擊展開第五章完整原文 📄`。

## 五、互動與動態體驗設計

1. **Scroll-Driven Entrance Animation (滾動漸入)**
   - 使用 `IntersectionObserver` 監聽 `.scroll-fade-in` 類別，當卡片或時間軸節點進入視窗時，自動加上 `opacity: 1; transform: translateY(0);`，未進入前為 `opacity: 0; transform: translateY(20px);`。
2. **Smooth Height Transition Accordion (平滑手風琴)**
   - 傳統的 `<details>` 展開時會瞬間跳躍，本案將以輕量 JS 控制折疊面板的 `max-height` 與 `opacity`，實現絲滑的滑動展開與收合效果。
3. **Interactive Step Progress (步進器回饋)**
   - 當使用者勾選步驟時，該步驟卡片的背景色與邊框會轉為淡綠色，並顯示已完成 (Checked) 圖示，為新手提供強烈的正向心理反饋。

## 六、實作與部署步驟

1. **實作**：根據本規劃書更新 `index.html`，引入重新設計的 UI 元件、SVG 圖示與響應式 CSS。
2. **測試**：
   - 驗證在 iPhone (375px) 與桌面端 (1440px) 的排版自適應與表格水平捲動是否正常。
   - 檢查所有折疊面板展開時的原文內容是否與 `guide.md` 逐字一致。
3. **部署**：
   - 將代碼推送到 GitHub 倉庫。
   - 於 GitHub Repository → Settings → Pages 啟用 Pages 服務，指定 `main` 分支根目錄 `/`。
   - 等待建置完成，開啟 `https://<你的帳號>.github.io/etf_beginner_guide/` 即可瀏覽極致體驗的網頁。
